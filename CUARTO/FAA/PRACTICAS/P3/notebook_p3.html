Practica realizada por Tomas Higuera Viso y Alejandro Naranjo Jimenez

Estructura del clasificador¶

En esta practica se nos pide desarrollar un clasificador que funciona con un algoritmo genetico. El algoritmo lo desarrollaremos utilizando el algoritmo de Pittsburgh. En este algoritmo cada cromosoma tiene un numero aleatorio de reglas. Cada una de estas reglas contiene los genes de cada atributo.

La estructura del clasificador implementada es la siguiente:

[ClasificadorAG] -- lista de -->[Cromosoma] -- lista de -->[Regla] ==> (Regla binaria o Rega enteros)

A) Generacion de la poblacion incial con especial indicacion del numero de reglas por individuo consideradas:¶

La poblacion incial se crea de manera aleatoria siempre dentro de los intervalos especificados para cada atributo. Las reglas inicales tienen una alta probabilidad de que los genes valgan 0, a pesar de ser creadas de manera aleatoria. De esta manera nuestro algoritmo empieza descubriendo los atributos menos relevantes y creara reglas con valor 0 en ellos.

El numero de reglas incial es aleatorio pero con un maximo pasado como parametro.

B) Mecanismo de cruce implementado:¶

El mecanismo implementado es el de cruce en un punto, es decir, se cruzan las reglas entre dos cromosomas, el punto de cruce es siempre aleatorio.

C) Mecanismo de mutacion implementado:¶

Mutamos un porcentaje aleatorio de individuos. Cada individuo elige de manera aleatoria una de las reglas y la muta. El la mutacion, dada una probabilidad, se muta cada gen, es decir se inserta de nuevo de manera aleatoria.

Detalles del codigo¶

ClasificadorAG¶

El clasificador tiene 2 funciones principales:

- entrenamiento(self, datostrain, diccionario): Usa el algoritmo genetico para evolucionar una poblacion hasta un numero maximo de generaciones o hasta un fitness determinado.

- clasifica(self, datostest, diccionario): Una vez se ha entrenado se escoge el mejor individuo y se clasifican los datostest con este.

Una vez clasificado y obtenido el array de predicciones hayamos las tasas de error haciendo uso de la funcion error(self, datos, pred).

La evolucion de la poblacion que se realiza en el entrenamiento hace uso de 3 funciones:

- seleccion_progenitores(self): Hace uso de la poblacion del clasificador para devolver otra poblacion del mismo tamaño proporcional al fitness.

- cruce(self, poblacion): Utilizando la poblacion recibida como parametros, con una proporcion pc, se escoge un numero par de individuos de manera aleatoria y de dos en dos cruzamos e insertamos vastagos en la nueva poblacion.

- mutacion(self, poblacion): Igual que en el cruce, cogemos n individuos y los mutamos.

Cromosoma¶

Los individuos de la poblacion son cromosomas que tienen un fitness y una lista aleatoria de reglas.

- predicciones(self, datos): Cuando una regla se cumple se incluye su clase en la votacion, sino se cumple no. 

-  calculate_fit(self, datos): Usa la funcion anterior y compara lo que predice el cromosoma con la clase de la familia. El fitness es el porcentaje de aciertos respecto a los errores.

Ademas, tambien tiene las funciones de cruce y de mutacion:

- cruce(self, progenitor_2): Elige un punto aleatorio del cromosoma y se realiza el cruce. Devuelve los dos vastagos.

- mutacion(self, pmbit, n_intervalos): De forma aleatoria selecciona una regla aleatoria y la muta.

Regla¶

Cada gen almacena los intervalos que almacena para un atributo. Existen dos tipos de reglas, las binarias y las enteras, la unica diferencia entre estas es la forma de representar el gen.

En las reglas de enteros cada gen contiene el numero del intervalo que acepta a 0.

En las reglas binarias, cada gen contiene una lista de 0s y 1s para indicar si se acepta un intervalo o no.

- seCumple(self, fila): Recibe una fila de datos transformada a regla y compara cada gen.

- mutacion(self, pmbit, n_intervalos): Se recorre cada gen y con una probabilidad pmbit el gen escogido se rellena aleatoriamente.

Instrucciones del calsificador¶

Hemos creado una funcion main para ejecutar el clasificador dentro del fichero ClasificadorAG.py. Esta funcion recibe como parametros el fichero de datos sobre el que se ejecutara el algoritmo, asi como el numero maximo de individuos en cada poblacion y el numero maximo de generaciones que se crearan.

Los diferentes parametros del clasificador son los siguientes:

ClasificadorAlgoritmosGeneticos(self,dataset=None, esBinaria = False, npoblacion=10,max_reglas=40,c_parada=97, max_g = 100,pc=85,pm=10,pmbit=5,pe=2,test=None)¶

Parametros:

- dataset: El conjunto de datos.
- esBinaria: El tipo de regla a utilizar.
- nPoblacion: Poblacion del algoritmo genetico.
- max_reglas: El numero maximo de reglas posibles.
- c_parada: El valor maximo de fitness como condicion para parar la  creacion de generaciones.
- max_g: El numero maximo de generaciones a crear.
- pc: La probabilidad de cruce.
- pm: La probabilidad de mutacion.
- pmbit: La probabilidad de mutar un gen usado en reglas.
- pe: La proporcion de individuos que se mantendran de una generacion anterior.
- test: El nombre del archivo donde insertar los datos para realizar plot.

En la ejecucion hemos utilizado como estrategia de particionado la validacion simple, con una solo particion. Para cambiar el tipo de particion tan solo tendremos que cambiar dentro de la funcion main el tipo de particion haciendo uso de las funciones de EstrategiaParticionado.py.
